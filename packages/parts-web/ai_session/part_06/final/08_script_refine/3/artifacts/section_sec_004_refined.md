# 平台技术架构

平台技术架构自下而上地展示了数据流转与价值提炼的过程，分为数据采集、模型分析和上层应用等多个层次。

## 1. 架构分层

### 1.1. 底层数据采集

基于成熟的操作系统级监控解决方案，实现全面、低开销的数据采集。

- **核心特点**：
  - **数据准确**：确保监控数据的精确性。
  - **低性能开销**：经过深度优化，对系统性能影响极小。

- **监控范围**：覆盖硬件、操作系统、JVM 容器、应用程序及动环系统。

- **数据维度**：全面覆盖系统级与应用级数据，包括系统、服务、进程、分区、网络等。

### 1.2. 模型抽象与实时分析

将采集的底层数据抽象为三大核心模型：

- **系统运行模型**
- **系统行为模型**
- **资产告警模型**

基于这些模型与原始数据，平台进行实时的告警分析，包括系统告警、行为告警和智能告警，并利用**根因分析（RCA）智能体**进行深度诊断。

### 1.3. 上层运维工具

平台提供一系列开箱即用的运维（O&M）工具，包括：

- 基线核查
- 系统巡检
- 智能对话
- 调优自愈
- 漏洞扫描与修复

## 2. 高可用网络架构

为保障平台自身的可靠性与高性能，设计了高可用网络架构，其核心机制包括：

- **多区域部署**
- **负载均衡**
- **冗余链路**

该架构确保了监控数据的稳定采集与控制指令的可靠下发，为上层智能应用提供坚实的网络基础。

## 3. 大模型集成能力

平台已具备灵活的大模型（LLM）接入能力，目前支持的主流模型包括：

- **电网行业模型**：
  - 国网：光明大模型
  - 南网：大问头模型
- **通用主流模型**：
  - 阿里通义千问
  - DeepSeek
