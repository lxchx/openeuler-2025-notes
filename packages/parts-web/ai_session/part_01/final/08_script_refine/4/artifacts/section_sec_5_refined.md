# 实践：HM IntelliCore决策引擎

## 1. 核心架构创新

IntelliCore决策引擎的核心架构相较于传统架构，引入了以下关键创新，以满足临床场景下对实时性和灵活性的高要求：

- **文本转规则 (Text-to-Rule)**：利用大模型将自然语言描述的业务逻辑转化为可执行的通用规则，以应对实时提醒等场景。
- **文本转SQL (Text-to-SQL)**：将自然语言转化为SQL查询，以满足报表类需求。
- **GoRules通用规则引擎**：一款全新开发的、不依赖特定业务的通用规则引擎。

## 2. “文本转规则”技术详解

### 2.1 面临的挑战

在临床应用中，决策提醒对实时性要求极高。直接使用大模型进行推理，其速度无法满足临床的实时计算需求。

### 2.2 解决方案

为解决此问题，我们采用了“模型生成规则”的策略，其核心是开发了一套“文本转规则”引擎服务。

1.  **模型分析与规则生成**：该服务集成了一个微调后的语言模型。当输入一段描述决策场景的自然语言文本时，模型会进行分析，判断其所属的决策类型，并最终生成一种通用的、脱离具体业务的规则产物（Rule Artifact）。

2.  **通用引擎执行**：生成的规则产物可被下游的通用规则引擎直接执行。

## 3. GoRules 通用规则引擎

基于上述“文本转规则”生成的产物，我们开发了一款名为GoRules的通用规则引擎。

### 3.1 设计理念

- **业务解耦**：引擎完全独立于原有的业务系统和规则引擎，不依赖任何业务属性。
- **输入/输出模型**：基于清晰的输入（数据）和输出（计算结果）概念进行设计。

### 3.2 核心特性

- **执行逻辑**：接收输入数据和指定的规则（Rule），即可产生计算结果。
- **内置算子**：引擎内部集成了大量算子（Operators），以支持复杂的计算场景。

### 3.3 价值与优势

- **降本增效**：无需再为不同业务维护大量的静态规则引擎，显著降低了开发和维护成本。
- **业务扩展性**：能够支持更多、更灵活的业务场景。

## 4. 应用场景与商业化

IntelliCore决策引擎已在多个场景成功落地。

- **国内应用场景**：
  - 智能推荐
  - 推荐评估
  - 推荐检查
  - 推荐治疗

- **国外应用场景**：
  - 主要集中于用药相关的推荐应用。目前，该方向已进入商业化落地阶段。
