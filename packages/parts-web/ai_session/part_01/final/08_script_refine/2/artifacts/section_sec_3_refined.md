# 破局：新一代CDSS智能引擎构想

## 1. 传统CDSS架构的困境

传统的临床决策支持系统（CDSS）普遍面临孤岛式、功能堆砌的困境。其架构通常呈现为“堆烟囱式”的形态，具体挑战如下：

- **架构孤立**：各业务系统与背后的支持模块（如NLP、模型、规则库）相互独立，形成数据和功能孤岛。
- **功能堆砌**：系统倾向于不断叠加新功能，而非进行智能化、一体化的融合。
- **依赖静态规则**：核心逻辑高度依赖静态规则引擎，难以应对复杂的临床场景。
- **数据处理瓶颈**：在处理来源多样、结构各异的异构数据时面临挑战。
- **硬件利用率低**：未能充分利用现代硬件（如GPU/NPU）进行计算加速。

## 2. 新一代智能引擎构想

为突破上述困境，我们提出新一代智能引擎的构想，旨在将“功能堆砌”的场景转变为“智能融合”，将“堆烟囱式”的架构升级为“一体化架构”。

核心目标包括：

- **替代静态规则引擎**：采用更灵活、智能的模型驱动决策。
- **解决异构数据处理**：建立统一的知识处理框架，高效整合不同来源的临床数据。
- **利用硬件加速**：充分利用NPU等硬件资源，对模型进行加速，提升响应速度与处理能力。

## 3. 实践案例与落地

基于以上构想，我们已在多个场景中进行了实践落地。

### 3.1 openEuler Intelligence CDSS知识库

该知识库主要面向四级诊疗场景，旨在让临床知识能被医护人员更高效地使用。相较于传统基于Elasticsearch架构的知识库（仅通过分词搜索，返回信息庞杂），新知识库提供了更精准、更具应用价值的知识服务。

### 3.2 惠每IntelliCore 智能决策核心

此架构主要应对五、六、七级等更高级的智能决策场景。其核心能力是在患者的诊疗全流程中，基于海量的病历信息进行智能决策。这些信息包括：

- 入院记录
- 病程记录
- 手术记录
- 检验、检查报告
- 各项指标与生命体征

当医护人员开立医嘱或进行检查时，系统能根据患者的实时情况进行分析，对可能存在问题的操作进行提醒。这不仅提升了医疗质量，也是降低医患纠纷、体现CDSS核心价值的关键。

### 3.3 基于昇腾的NPU模型加速

为配合国产化硬件设备，我们利用昇腾（Ascend）社区的开源模型与镜像，实现了Embedding模型的NPU加速。此外，我们也在探索vNPU（虚拟NPU）切分技术，该技术能将一张物理NPU卡虚拟化为多张逻辑卡，实现资源的隔离与高效利用，避免多任务场景下的资源抢占，从而达到更稳定的模型加速效果。
