## 下一步工作方向

基于客户反馈与实际落地中的痛点，我们梳理了下一步的重点工作方向，旨在提升编译器性能与易用性。

### 1. 自动图融合

- **现状**：依赖基于经验积累的手工融合策略。
- **目标**：结合已有的人工调优经验，共建更通用的自动图融合策略。
- **价值**：提升融合能力的泛化性，灵活覆盖更多场景的融合诉求。

### 2. 增强对动态 Batch 的支持

- **现状**：当前编译器仅支持静态 Shape。
- **痛点**：
  - 面对变长输入（动态 Batch）时，会重复触发编译。
  - 该过程不仅增加了编译耗时，还会生成大量冗余的编译结果。
- **目标**：提升编译器对动态 Batch 的支持能力。
- **价值**：更好地适应真实业务场景中的流量波动。

### 3. 优化算子间并行执行

- **现状**：图算融合将多个小算子融合成单个大算子，减少了调度开销，但也降低了 `inter-op` 级别的并行性。
- **目标**：解决融合带来的并行性下降问题，让融合后的大算子也能在多核 CPU 中并发执行。
- **价值**：充分发挥多核 CPU 的并发性能。

### 4. 构建自动 PTQ 量化工具链

- **目标**：计划打造一套面向 CPU 的自动 Post-Training Quantization (PTQ) 工具链。
- **应用场景**：重点服务于高精度的搜索与推荐场景。
