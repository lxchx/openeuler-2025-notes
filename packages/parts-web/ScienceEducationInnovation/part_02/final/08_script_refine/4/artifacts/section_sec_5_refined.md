# PM层参数存储：PetPS

## 1. 背景：传统预取方法的局限性

在推荐模型的特定应用场景中，传统的预取（Prefetching）方法存在适用性问题，难以满足性能要求。

## 2. PetPS：面向持久性内存的解决方案

为应对上述挑战，我们设计了 PetPS 系统，一个专为持久性内存（Persistent Memory, PM）设计的参数存储方案。

### 核心机制

- **卸载聚集操作**：PetPS 的核心思想是利用网卡（NIC）的直接内存访问（DMA）引擎来卸载（Offload）参数的聚集（Gather）操作。
- **规避高延迟开销**：通过卸载操作，系统有效规避了因持久性内存高延迟特性而导致的 CPU 高昂的访存开销。
- **解决一致性挑战**：该方案同时解决了在持久性内存上进行参数存储时的一致性问题。
